# vba-canned-responses-and-automations

Hello,

My name is Wyatt and this project is a sample of work I did while working at a logistics company. Please be aware that all data has been anonymized and many parts redacted to ensure data privacy standards. The sample data does not represent any historical business data and is only used to demonstrate the effectiveness of the coded solution.

The original problem to be solved involved slow response times to truck drivers causing unnecessary friction among the team. Drivers felt like the management team were slow and lazy while managers felt overwhelmed by the volume of messages and calls coming in. As more messages came in, management slowed down in their overall response times which made drivers feel like they needed to call in more often. The more calls that drivers sent in further reduced manager effectiveness as they would not be able to receive multiple calls at once, further reducing the total number of accepted calls.

The task was to generate canned responses to speed up resonse times to common inquiries to reduce the immediate need for drivers to call in as frequently as well as provide automations for common actions that took a considerable amount of time. In practice, following implementation of these improvements, weekly message responses and calls received doubled as common messages received immediate attention opening time to be able to intake more calls throughout the day. The average benchmark per manager was ~100 received calls and ~3,000 message responses per week. With canned responses and automations, average received calls rose to 200+ and average message responses per week jumped to 7,500+.

To explore this project, please take a look at the "Functions.vba" file noting ~2,000 lines of code developed over the course of ~2 years. The original functions were built off of a previous connector with an internally developed program which many of the connectors have been redacted to retain data privacy standards.
